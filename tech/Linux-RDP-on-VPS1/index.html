VPS主机中一键安装Linux桌面环境(脚本)和RDP远程桌面连接

Feb 29, 2020

Category▸  Tech
Tag▸  VPS   RDP



Source: 搬运

  • 链接：https://wzfou.com/linux-zhuomian/
  • 一键脚本项目：
    • 项目：https://github.com/MeowLove/Linux-Remote-Desktop-Environment
    • 备份：https://github.com/freehao123/Linux-Remote-Desktop-Environment
  • 一键脚本安装命令： 
      #下载脚本
  wget --no-check-certificate -qO ~/Linux-Remote-Desktop-Environment.sh 'https://raw.githubusercontent.com/freehao123/Linux-Remote-Desktop-Environment/master/Download/Common/Load/Linux-Remote-Desktop-Environment.sh' && chmod a+x ~/Linux-Remote-Desktop-Environment.sh
    
  #CentOS 7+安装 (sudo yum install wget)
  bash ~/Linux-Remote-Desktop-Environment.sh -CentOS
  #Debian 9+安装
  bash ~/Linux-Remote-Desktop-Environment.sh -Debian
  #Ubuntu 18.04+安装 (sudo apt-get install wget)
  bash ~/Linux-Remote-Desktop-Environment.sh -Ubuntu

 

Issue: 多种尝试始终port:3350相关的xrdp连接失败

  • 尝试多平台，相同issue
    • 使用CentOS 7/8/9、Debian 8/9/10、Ubuntu 18/19，均安装过，均出现 “port:3350相关的xrdp连接失败” 错误。
    • 参考了网上的解决方案 1、2，仍然无效。
  • 因此思考是否与一键脚本中连接的项目网站有关？ 测试一次通过。

 

Solution: 更换项目网站

  • 亲测配置: Debian 9, 617M@Momery,10G@Storage,windows rdp成功连接，但非常卡。建议改用更优配置(如：Debian 9, 3.75G@Momery,30G@Storage)。
  • 脚本中的项目更换为原始网址 
      wget --no-check-certificate -qO ~/Linux-Remote-Desktop-Environment.sh 'https://raw.githubusercontent.com/MeowLove/Linux-Remote-Desktop-Environment/master/Download/Common/Load/Linux-Remote-Desktop-Environment.sh' && chmod a+x ~/Linux-Remote-Desktop-Environment.sh
    
  #Debian 9+安装
  bash ~/Linux-Remote-Desktop-Environment.sh -Debian
  • 安装过程中，仅会出现一次关于”keyboard configuration”的互动图像界面，直接回车即可，自动返回终端界面继续安装，直至完成。

  • 完成后，windows PC端使用内置的远程桌面连接直接连接；手机端，使用微软公司发行的Microsoft Remote DesktopApp即可连接。
