在安卓手机上使用ADB & Fastboot命令

Jan 7, 2020

Category▸  Tech
Tag▸  Android


背景：
当一部安卓手机需要透过adb命令进行USB调试时，默认只能连接PC，在外时非常不便。该方法可成功使用”手机#1”代替”PC”，使用ADB & Fastboot命令，连接&操作”手机#2”。

好处：
“手机#2”不必root情况下，可随时透过adb方式，最大化的管理手机相关权限（如可使用App Ops、Greenify）。


主要参考链接

  • Use ADB & Fastboot Commands Without a Computer, BY STEPHEN PERKINS, 07/17/2019

 

重点摘录

  • “手机#1”要求

    • 已root
    • 已安装Magisk，安装模块ADB & Fastboot for Android
    • 需支持USB OTG连接
    • 安装命令行终端，如Termux

 

  • “手机#2 (non-root)”要求

    • 打开”开发者模式”下USB调试

 

实测分享和注意点

  • Cable连接两部手机

    • 亲测发现，当cable两端无需转换接口时，文件传输和USB调试均可正常工作；但当cable两端(或任一端口)需转换接口时，仅文件传输可正常工作，USB调试无反应。因此，建议使用无需转换接口的cable直接连接两部手机。

 

  • 操作步骤

    1. 手机#1：将USB OTG数据交换功能打开。
    2. 手机#2：将USB调试打开。
    3. 连接cable。
    4. 手机#2：在弹出的”USB偏好设置”中，选择USB受控于 - 此设备(如果有该选项)；选择USB用途 - 不进行数据传输。
    5. 手机#1：打开Termux，获取su后，输入adb devices。
    6. 手机#2：弹出允许USB调试吗？窗口，打勾一律允许使用这台计算机进行调试&点击允许。
    7. 手机#1：当第6步骤”被允许”后，”手机#1”SDcard根目录下自动生成两个文件 - adbkey、adbkey.pub。
    8. 手机#1：Termux获取su后，输入adb devices后，当看到”手机#1”的设备号时，即成功了。进而，输入需要的adb命令。
    9. (optional) 手机#2：安全起见，完成相关操作后，将USB调试关闭。撤销USB调试授权。

  • ADB Commands

    • 启动shizuku
      • 手机每次重启后或者因”USB调试”关闭而失效，都需要再次启动shizuku (当”shizuku”未启动时，App Ops不可进入设置，但”App Ops”已设置过的应用权限，永久有效)。 
          adb shell sh /sdcard/Android/data/moe.shizuku.privileged.api/files/start.sh

     

    • greenify  (一次性设置即可)
      • Accessibility service run-on-demand: 
          adb -d shell pm grant com.oasisfeng.greenify android.permission.WRITE_SECURE_SETTINGS
      • Aggressive Doze on Android 7.0+ (non-root): 
          adb -d shell pm grant com.oasisfeng.greenify android.permission.WRITE_SECURE_SETTINGS
      • Doze on the Go: 
          adb -d shell pm grant com.oasisfeng.greenify android.permission.DUMP
      • Aggressive Doze (on device/ROM with Doze disabled): 
          adb -d shell pm grant com.oasisfeng.greenify android.permission.DUMP
      • Wake-up Tracker: 
          adb -d shell pm grant com.oasisfeng.greenify android.permission.READ_LOGS
      • Background-free enforcement on Android 8+ (non-root): 
          adb -d shell pm grant com.oasisfeng.greenify android.permission.GET_APP_OPS_STATS

     

    • full backup/restore all Android device’s data w/o root，详解
      • full backup (can encrypt your backup with a password) 
          adb backup -apk -shared -all -f <filepath>/backup.ab

        另，网络上建议：backing up music, pictures, video, and other internal storage items manually；SMS自行单独备份。

      • full restore 
          adb restore <filepath>/backup.ab

     

    • Island 文件传送门 - File Shuttle  (一次性设置即可) 
        pm grant com.oasisfeng.island android.permission.INTERACT_ACROSS_USERS
