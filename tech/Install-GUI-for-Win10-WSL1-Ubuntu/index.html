微软Subsystem子系统WSL - Ubuntu安装图形用户界面

Mar 29, 2020

Category▸  Tech
Tag▸  WSL



背景：Windows 10 已安装 WSL（windows subsystem for linux）- Ubuntu 18.04。

  • Subsystem Path on Windows 10: 
    C:\Users\<我的用户名>\AppData\Local\Packages\CanonicalGroupLimited.Ubuntu18.04onWindows_79rhkp1fndgsc\LocalState\rootfs

 

首次安装WSL1步骤：

  • Step1: Powershell执行如下命令&重启操作系统 (作用等同于”启用或关闭Windows功能”)

      Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

  • Step2: 在”应用商店”中，安装想要的Linux发行版

    • 安装完成后，需进入Linux Terminal进行初始设置(用户名+密码)。

 

安装Linux图形用户界面通常有两种方法：

  • 使用X-Windows安装使用图形化界面

  • 通过本机远程桌面连接 (本帖使用方法)

    1. 查看当前系统版本

      lsb_release -a

    2. 查看运行ubuntu是32位或是64位

      #如果显示i686, 则表示32位；如果显示 x86_64, 则表示64位
  sudo uname --m

      #其他命令
  sudo uname --s     #显示内核名字s
  sudo uname --r     #显示内核版本
  sudo uname --n     #显示网络主机名
  sudo uname --p     #显示cpu

    3. 替换成国内源/etc/apt/sources.list

      deb http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse  
  deb http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse  
  deb http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse  
  deb http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse

  deb http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse  
  # 源码

  #deb http://extras.ubuntu.com/ubuntu/ bionic main

    4. 日常更新&准备工作

      sudo apt update
  sudo apt upgrade
	
  #或者
  sudo apt update && sudo apt upgrade

    5. 使用”本机远程桌面连接”方式：在Ubuntu Terminal中的安装步骤

      #安装xorg（包括显卡驱动、图形环境库等等一系列软件包）
  sudo apt install xorg

  #安装xfce4（运行在类Unix操作系统上，提供轻量级桌面环境）
  sudo apt install xfce4

  #安装xrdp（一种开源的远程桌面协议（RDP）服务器）
  sudo apt install xrdp

  #配置xrdp（配置端口）, 默认端口号为3389，此处配置为3399
  sudo sed -i 's/port=3389/port=3399/g' /etc/xrdp/xrdp.ini

  #向.xsession中写入xfce4-session
  sudo echo xfce4-session >~/.xsession

  #重启xrdp服务
  sudo service xrdp restart                  #停止xrdp服务： sudo service xrdp stop

    6. 在windows远程桌面登录xfce4图形界面

    • 在微软远程桌面连接(运行mstsc)，点击进入，计算机栏输入【本机IP:端口】（即：127.0.0.1:3399(for WSL1), localhost:3399(for WSL2)），用户名栏输入子系统用户名，点击连接。
      • NOTE: 每次重启Windows系统后，需要在WSL Terminal中"sudo service xrdp restart"重启xrdp服务一次，否则出现"远程连接失败"错误。
      • 在”微软远程桌面连接”窗口，”显示/体验”Tab中均可做呈现设置。
        • 如：显示配置 - 2560x1440，颜色 - 真彩色24位，体验 - LAN/不勾选”动画、视觉样式”。

     

    • 进入xfce4桌面，基本设置 (一次设置永久生效)：
      • 『Screensaver』：开始菜单/应用/Screensaver: no Screensaver

       

      • 『Settings from StartMenu』：开始菜单/应用/Seettings: Setting Manager (桌面窗口样式/显示字体大小等等)
        • Appearance: “Fonts” Tab, Size：18 (主要的字体大小)
        • Desktop: “Icons” Tab, Icon Size：78 (桌面图标大小)
        • File Manager: “Side Pane” Tab, 设置左侧图标大小，Shortcut Pane选择Normal，Tree Pane选择Small;窗口大小直接鼠标拉伸；文件夹图标大大小，直接通过右击鼠标选择zoom in或者zoom out。
        • Panel: Panel 1 Row Size：41; Panel 2 Row Size：64 (上下两个面板条的长度设置)
        • Windows Manager: Style/Default/Title Font: Size 18 (设置窗口界面中的文字大小)
        • Orange Perference: 点击打开即可 (日历，首次登录需打开一次即可，以后无需)
        • Workspace: layout/number of workspaces: 1~4，选择自己需要的数字 (桌面右上方，可右击删除workspace)

       

      • 『Terminal设置』: 打开Terminal窗口
        • Tab: “Preference” -> Side Panel: “profile/unamed/” -> “text appearence” Property:
          • Initial Terminal Size: 144 columns, 40 rows
          • Customer Font: 19
        • 关闭&重新打开Terminal，设置即已生效。

     

    7. 『可选』在子系统中安装常用程序

    • Install Dukto

        #下载并安装.deb包
  $sudo apt install ./dukto_6.0-1_amd64.deb

    • List Installed Packages on Ubuntu

        sudo apt list --installed
  sudo apt list --installed | less
  sudo apt list --installed | grep package_name

  sudo dpkg-query -l | less
  sudo dpkg-query -l | grep package_name	   

  sudo dpkg-query -f '${binary:Package}\n' -W > packages_list.txt    #将已安装包列表保存在"packages_list.txt"
  sudo xargs -a packages_list.txt apt install                        #安装"packages_list.txt"中含列的安装包

  sudo dpkg-query -f '${binary:Package}\n' -W | wc -l                #统计已安装的安装包的数量

    • Windows<->WSL File Transfer

        mv /mnt/c/download/file1 /home/Downloads  #从Windows C盘移动文件至WSL
  cp /mnt/c/download/file2 /home/Downloads  #从Windows C盘复制文件至WSL；"-r"为复制目录；"-f"为强制，需逐个确认。
