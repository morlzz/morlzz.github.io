Hyper-V 虚拟服务器

Jul 12, 2019

Category▸  Tech   搬运
Tag▸  OS

Abstract ▹ Hyper-V 常见问题,微软官方安卓虚拟器


1. 虚拟OS过程中，出现”死机”状态

安装或启动过程中，出现”死机”状态，Hyper-V关闭也没有用

解决方法 (原出处)：

  • 使用微软官方工具dgreadiness。
  • 解压后, 在Powershell中运行：.\DG_Readiness_Tool_v3.6.ps1 -Disable。
  • 重启后问题则解决了。


2. Visual Studio Emulator for Android

微软官方提供，在windows上的安卓模拟器，借助”Hyper-V” (即需事先启用 Hyper-V)。

主机、虚拟机之间的网卡设定注意事项：

  • Hyper-V虚拟服务器 “虚拟交换机管理器”配置中，确保有三块网卡：
    • 默认：Default Switch，不管它；
    • Windows Phone Emulator Internal Switch，一般也是默认安装的，如没有自建一个，名字随意，连接类型选内部网络。
    • 需自建一个”外部网络”，用来绑定”主机的物理网卡”，同时需允许共享该物理网卡。

 

  • 安卓虚拟机 配置中，需确保设定两块网卡：
    • 必须设定两块网卡：Internal Switch (内部)、External Switch (外部，绑定主机的”物理网卡”，允许该网卡共享)；
    • Internal Switch 顺序在前，External Switch 顺序在后 (未颠倒过来测试，国外某论坛上这么说’the order does matter’；因默认是这个先后顺序，故未亲测)。

 

  • 安卓虚拟机 虚拟网卡的工作方式：
    • “安卓虚拟机外部网络，”绑定”主机的物理网卡 X“，则在主机”网络适配器”中，自动安装”X的虚拟网卡 Y” (Y，被安卓虚拟机配置成网卡)；
    • 自动生成”网桥”，桥接X，至此，”X” 已无法被主机使用；但透过桥接，主机上网不受影响；
      • 但主机的SMB受到影响，即手机无法通过SMB访问到主机；
      • SMB问题解决方法：主机上使用”双网卡”，除了以上网卡X用于与Hyper-V桥接，另一”网卡K“用于主机日常的使用；在主机上，可同时使这两个网卡连接网络。(注意：实际操作时，先禁用”网卡K“，待网卡X全部配置成功后，再启用”网卡K“。)

     

    • 每次启动”安卓虚拟机”时，被问及’是否重配置网络时’，选择忽略/否。

 

  • 其他问题：
    • TunSafe (Wireguard) 不能用
      • “Hyper-V虚拟服务器”和”安卓虚拟机”配置中，删除 Wireguard涉及的网卡 TAP-Windows Adapter V9，即可。
