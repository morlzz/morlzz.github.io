Arba Wallet 功能、特点、缺点

 

特点：

  • 便利性：
    • 30种数字货币 + 52种法币。
      • 支持 native 四种数字货币：btc/bch/lte/eth (即支持这四种的原生钱包地址) 。
      • 其他26种数字货币和52种法币: 通过app内的exchange“合成”对应币种。
      • 所有exchange，实则采用“smart contract”，即锁定等值的原native数字币种。
        • 官网案例：btc通过”euro contract”转换成euro，之后用户想将euro转换成eth，实际过程是 “convert contract”：”redeem euro contract” first -> “fix value of btc” to the amount of eth = “create eth contract”.
        • smart contract：保存在区块链，因此arba需要支付network mining fee，arba将此费用折摊到exchange rate转换率中。
          • 即，”arba spread on exchange rate”包含两部分：1) 实际发生的 network fee; 2) abra宣称唯一用来盈利的转换率差价(由arba自己设定)。


 

  • 数字货币<–>法币：
    • 目前，美国和菲律宾，可以自由在”数字货币、法币”之间, deposit、withdraw from/to 传统银行。
    • 其他国家，可通过abra合作伙伴simplex，使用信用卡直接购买btc, simplex收取5%手续费(min. 10usd)。

 

  • 关于安全性：
    • 私钥/recovery phrase 完全由用户掌握(non-custodial wallet)，arba 无法得知。
    • HD wallet，每次deposit数字货币后，钱包地址变自动变更，好处：保护隐私，(旧的依旧生效)。

 

缺点：

  • 使用其他开源数字钱包导入Abra phrase时，失败，可见，Abra私钥/recovery phrase并非标准的phrase，可能有加入校验码，如果将来Abra关闭了App，存入的数字资产会比较麻烦。
  • 币种间的转换费颇高，~2.5%。
  • 市场价和Markup后的价格，没有一个直观的对比，使得客户很难有一个直接的了解，个人觉得这是一点人为的"刻意不透明"。
  • 绑定的美国银行账户限制颇多：1) 支持银行少；2) 需要没有SSN或者税号。


Information from Abra Official

 

1. All Transactions Must Confirm Via Blockchain

  • All underlying currency is BTC, and all transactions must confirm via blockchain.
  • Abra uses Bitcoin-based contracts to facilitate cryptocurrency and fiat investments on the blockchain.
  • This network fee is required to ensure your transaction is confirmed by the Bitcoin mining network.

 

2. Transaction Types

  • Send Funds (Scenario: Abra钱包之间 发送)
    • Send Fiat
      • Merely choose ‘send funds’ in the app, this would not be a withdrawal. Abra does not charge an Abra fee (inc. no network fee) to ‘Send funds’ to another Abra wallet.
      • But still must be confirmed by blockchain.
    • Send BTC/LTC/BCH/ETH
      • Be charged a network fee to withdraw BTC/LTC/BCH/ETH from an Abra wallet and send to another Abra wallet.
      • A withdrawal will encounter a network fee.
    • Other Cryptocurrency Cannot Be Sent

 

  • Withdrawal (Scenario: Abra钱包->其他数字钱包 提现)
    • A withdrawal will encounter a network fee
    • Abra currently support BTC/LTC/BCH/ETH withdrawal

 

  • Withdrawal (Scenario: Abra钱包->Bank 提现)
    • Abra currently support BTC/LTC/BCH/ETH withdrawal; A withdrawal will encounter a network fee.
    • Support Fiat Transfer, no Abra fee (inc. no network fee); The charge fee depends on bank/third-party their own policy.
    • All above case must be confirmed by blockchain.

总结

  • Send/Transfer Fiat: no Abra fee (inc. no network fee);
  • Send/Transfer/Withdraw BTC/LTC/BCH/ETH: The actual transaction type is “A withdrawal”, a withdrawal will encounter a network fee.
